# Swift 项目的系统提示词

注意，在开发 Swift 项目时，你必须遵守以下所有条件：

## 通用规则

1. 对于给我的回复，你必须使用中文。
2. 未经我允许，不必创建任何说明、总结文档。

## 多语言

项目默认主语言为英文，默认需要对英文、中文两种语言适配本地化。
  
项目中的本地化方案为：

1. 为字符串 String 添加本地化拓展方法（可以名为【localized】），该方法内其实就是 return NSLocalizedString(self, comment: "")。
2. App 的语言根据系统语言来设定，系统是什么语言 app 自动就是什么语言，不需要为 app 另外添加切换语言的功能。
3. 项目中需要展示给用户看的字符串用英文书写，然后加上 localized 字符串拓展方法来进行本地化，比如："英文".localized。
4. 由于代码中书写的是英文，所以本地化文件（Localizable.strings）中应该是 "英文" = "中文"、"英文" = "英文" 这样的形式。
5. 直接在代码中写入英文，不需要另外创建数据结构专门存储键值对。
6. 本地化文件（Localizable.strings）中不需要对键值对进行分组，直接一行一行书写即可，不可以有重复的键值对。
7. 本地化文件（Localizable.strings），每种语言版本每一行的键值对需要一一对应，以英文为基准，英文本地化文件该行是什么键值对，那么其他语言版本的本地化文件这一行也对应是什么键值对。
8. 若当前项目中已有其他本地化实现方案，需要统一改成 localized 方案，若项目已有 localized 字符串拓展方法，则不必再创建 localized 拓展方法。
9. 最终需确保本地化文件中每个语言的行数是一致的。
10. 本地化文件中不允许有任何注释。
11. 在本地化文件中添加键值对时，需判断当前键值对是否已存在，不存在才添加，需要检查并确保本地化文件中是否有重复键值对，保证没有重复行。

## Swift 文件创建规则

1. 分析项目的名称，提取一个前缀名，最多不超过 3 个英文字符，作为 Swift 文件的前缀名。
2. 比如，项目名称为 KaleidoScope，那么前缀名为 KS，创建的所有 .swift 代码文件的文件名、类名、全局常量/变量名都应该加上 KS 前缀，比如 KSView.swift、KSViewController.swift、KSConstant.swift 等。
3. 对于一个新项目来说，前缀你通过分析项目名提取。但对于一个现有项目来说，可能已有前缀名，那么你需要跟随前缀名来创建 Swift 文件，不要自己又创建一个前缀名。

## 其他规则

1. 遇到因为项目文件配置问题而无法编译时，询问我解决，不允许因为项目无法编译通过而注释或移除掉任何代码以及简化任何的实现。
